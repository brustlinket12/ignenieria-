# Skill: README Aesthetic Enhancer

**Cuándo usar este skill:**  
Úsame siempre que el usuario pida mejorar, decorar, embellecer o "hacer más profesional" un README.md, AGENTS.md, informe de investigación o cualquier documentación Markdown. Especialmente útil para comparativas de herramientas, flujos de agentes de IA y documentos técnicos.

**Objetivo principal:**  
Transformar un README plano en uno **visualmente atractivo, moderno y profesional** sin perder claridad ni información.

### Reglas obligatorias (sigue siempre estas instrucciones)

1. **Mantén la información original**  
   No elimines ni resumas contenido importante. Solo mejora la presentación.

2. **Elementos visuales que debes agregar (en este orden recomendado):**
   - **Encabezado superior atractivo** (título grande + subtítulo + emojis o badge principal)
   - **Badges / Shields** en la parte superior (usa https://img.shields.io):
     - Última actualización
     - Versión
     - Tipo de proyecto (Research, AI Agents, etc.)
     - Licencia
     - Estrellas (si aplica)
   - **Tabla de contenido** con enlaces
   - **Divisores visuales** (`---` o `***` con emojis)
   - **Diagramas Mermaid** cuando sea relevante:
     - Flujo de trabajo de agentes
     - Comparativa (mindmap o flowchart)
     - Arquitectura de herramientas
     - Timeline
   - **Tablas elegantes** con alineación centrada donde corresponda y emojis en headers
   - **Secciones con iconos** (usa emojis relevantes: 📊 ⚡ 🔍 🛠️ 🎯 etc.)
   - **Bloques de código** con lenguaje correcto y títulos
   - **Imágenes o placeholders** para capturas de pantalla (con alt text descriptivo)
   - **Callouts / Alertas** (usando > o bloques de cita) para conclusiones clave
   - **Pie de página** con fuentes, última actualización y "Decorado con ❤️ por README Aesthetic Enhancer"

3. **Estilo visual recomendado (2026 best practices):**
   - Usa encabezados grandes (`#`, `##`, `###`)
   - Coloca badges en una sola línea al principio
   - Usa listas con emojis en lugar de guiones simples
   - Añade al menos 2-3 diagramas Mermaid por README largo
   - Mantén colores consistentes a través de badges y emojis
   - Asegura que todo sea responsive y se vea bien en GitHub dark/light mode
